## Release Trigger Rule

A new version is created when:

- ≥1 TASK marked as done
- system passes release-governor audit
- CHANGELOG updated

---

## Versioning Guide

PATCH (0.1.x)
- UI improvements
- small features
- bug fixes
- task completion

MINOR (0.2.0)
- new capability
- new flow
- structural change

MAJOR (1.0.0)
- stable production-ready system
